# AIGC 风险地图（阶段一）

面向文件：`thesis.tex`（仅主文正文）

## 一、结构层风险（按优先级）

1. 第5章《关键模块设计与实现》
   - 风险类型：技术路径“单线成功叙事”占比高。
   - 典型问题：流程写法稳定为“定义-机制-效果”，失败分支叙述不足。

2. 第6章《系统测试与结果分析》
   - 风险类型：结论先行句过多，样本波动与异常处理过程弱。
   - 典型问题：以“达标结论”收尾的段落密集，过程痕迹较薄。

3. 第7章《总结与展望》
   - 风险类型：对称总结模板（方法/工程/应用）重复。
   - 典型问题：表述完整但“口号化”较强，不确定性表达不足。

4. 第3章《系统需求分析》
   - 风险类型：判断句偏抽象，场景倒推过程较弱。

5. 第2章《相关研究与技术基础》
   - 风险类型：文献串联密度高，局部“教材式综述”痕迹。

6. 第1章《绪论》
   - 风险类型：仍存在标准学术模板段（问题-意义-目标）。

## 二、语言层风险

- 并列句型重复：连续“X 方面 / Y 方面”或“先结论后解释”的节奏较稳定。
- 总结句偏多：第6章与第7章节末结论句密度高。
- 不确定性覆盖不足：虽有“在当前样本下”等表达，但仍有确定性断言集中段。

## 三、内容层风险

- 过程细节不足：调参与配置淘汰过程叙述不充分。
- 失败案例不足：第5-6章成功路径多，故障场景和修复策略展开较少。
- 取舍理由不足：已有“为什么选”，缺“为什么不选备选”。

## 四、阶段二执行顺序

`第5章 -> 第6章 -> 第7章 -> 第3章 -> 第2章 -> 第1章`

## 五、修改边界

- 不改：章节标题、公式标签、表内关键数值、接口路径字符串、引用映射。
- 只改：表达层、过程层、案例层叙述，不改变结论方向。
