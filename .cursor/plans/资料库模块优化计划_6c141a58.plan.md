# 资料库模块优化计划

## 目标与范围
- 聚焦 `资料库` 模块的两类对象：学习资料（materials）与课程作业（assignments）。
- 解决三个问题：
  - 资料展示弹窗存在占位内容、信息结构不清晰。
  - 作业创建字段应明确为“标题 / 问题 / 截止时间”。
  - 已批改作业详情页应稳定为“左详情、右批改”，两侧独立滚动。

## 现状结论（基于现有实现）
- 前端核心在 [`E:/Workbench/毕设/ailearn/components/dashboard/course-library-page.tsx`](E:/Workbench/毕设/ailearn/components/dashboard/course-library-page.tsx)、[`E:/Workbench/毕设/ailearn/components/dashboard/create-library-item-dialog.tsx`](E:/Workbench/毕设/ailearn/components/dashboard/create-library-item-dialog.tsx)、[`E:/Workbench/毕设/ailearn/components/dashboard/library-page-dialog.tsx`](E:/Workbench/毕设/ailearn/components/dashboard/library-page-dialog.tsx)。
- 数据核心在 [`E:/Workbench/毕设/ailearn/prisma/schema.prisma`](E:/Workbench/毕设/ailearn/prisma/schema.prisma) 与 [`E:/Workbench/毕设/ailearn/app/dashboard/actions.ts`](E:/Workbench/毕设/ailearn/app/dashboard/actions.ts)。
- `Assignment` 当前缺少明确“问题”字段；批改结果主要塞在 `StudyRecord.meta`，结构化不足；`LearningMaterial` 在不同入口存在 `url/content` 写入不一致。

## 数据模型是否需要修改
- **需要，且建议最小增量修改**：
  - 在 `Assignment` 增加 `question` 字段（用于独立承载作业问题）。
  - 在 `StudyRecord` 增加结构化批改字段（如 `reviewStatus/reviewComment/reviewScore/reviewedAt/reviewerId`）。
- **资料模型短期不强制改表**：
  - 先统一写入协议（链接写 `url`、正文写 `content`、摘要写 `description`），避免现有 `LearningMaterial` 语义继续漂移。
- **迁移策略**：先“兼容新增 + 回填 + 双读”，稳定后再考虑收紧约束（例如 `question` 变必填）。

## 分步骤实施

### 阶段 1：数据层与类型层打底（低风险）
- 更新 [`E:/Workbench/毕设/ailearn/prisma/schema.prisma`](E:/Workbench/毕设/ailearn/prisma/schema.prisma)：新增 `Assignment.question` 与结构化批改字段。
- 生成 Prisma migration，并准备一次性回填脚本：
  - 将历史 `Assignment.description` 回填到 `question`（当 `question` 为空时）。
  - 从 `StudyRecord.meta.teacherReview` 回填结构化字段（尽力回填，不阻塞上线）。
- 更新前端类型定义 [`E:/Workbench/毕设/ailearn/components/dashboard/library-types.ts`](E:/Workbench/毕设/ailearn/components/dashboard/library-types.ts) 与聚合查询 [`E:/Workbench/毕设/ailearn/lib/dashboard-data.ts`](E:/Workbench/毕设/ailearn/lib/dashboard-data.ts)，实现“新旧字段兼容读取”。

### 阶段 2：创建流程优化（资料 + 作业）
- 改造创建弹窗 [`E:/Workbench/毕设/ailearn/components/dashboard/create-library-item-dialog.tsx`](E:/Workbench/毕设/ailearn/components/dashboard/create-library-item-dialog.tsx)：
  - 作业模式固定 3 字段：标题、问题、截止时间。
  - 截止时间改为更稳定的日期时间输入与前端校验。
  - 资料模式明确字段含义（标题、摘要、资料内容/链接），减少模糊输入。
- 统一创建动作 [`E:/Workbench/毕设/ailearn/app/dashboard/actions.ts`](E:/Workbench/毕设/ailearn/app/dashboard/actions.ts) 以及课程路由下相关 actions（teacher/student 目录）：
  - 作业写入 `question`，`description` 作为可选简述。
  - 资料写入协议统一，避免 `url/content` 混写。

### 阶段 3：展示弹窗优化（去占位、提可读性）
- 重构资料详情弹窗 [`E:/Workbench/毕设/ailearn/components/dashboard/library-page-dialog.tsx`](E:/Workbench/毕设/ailearn/components/dashboard/library-page-dialog.tsx)：
  - 删除占位文案（如示例链接、占位区说明）。
  - 建立稳定展示块：基本信息、资料正文、附件/链接区。
  - 教师与学生共用核心展示结构，仅按角色控制操作按钮。
- 优化列表辅助信息：必要时调整 [`E:/Workbench/毕设/ailearn/components/dashboard/library-table-header.tsx`](E:/Workbench/毕设/ailearn/components/dashboard/library-table-header.tsx) 以减少 `-` 带来的认知噪音。

### 阶段 4：已批改作业“左右分栏 + 独立滚动”强化
- 在 [`E:/Workbench/毕设/ailearn/components/dashboard/library-page-dialog.tsx`](E:/Workbench/毕设/ailearn/components/dashboard/library-page-dialog.tsx) 的教师作业详情中：
  - 左栏：作业详情（标题、问题、截止时间、学生答案、AI初评）。
  - 右栏：批改区（状态、评分、评语、批改时间）。
  - 两栏分别设置独立滚动容器，保证长内容互不干扰。
- 保留移动端降级策略：小屏可堆叠，但桌面端固定双栏体验。

### 阶段 5：联调与回归
- 覆盖关键链路：
  - 老师创建资料/作业 -> 列表展示 -> 详情查看。
  - 学生提交作业 -> 老师批改 -> 已批改详情查看。
- 重点验证兼容性：旧数据（仅 `description`、仅 `meta.teacherReview`）在新 UI 下可读。
- 执行 lint/typecheck 并修复引入问题。

## 验收标准
- 资料弹窗无占位内容，信息块清晰且师生可读一致。
- 作业创建仅暴露“标题、问题、截止时间”三要素，并正确落库。
- 已批改作业详情在桌面端稳定呈现左右两栏，且两栏独立滚动。
- 新旧历史数据都能正常展示，不出现空白或字段错位。
